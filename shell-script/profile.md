# プロファイルとは何か（/etc/profile）

## これは何？
/etc/profile は、Linuxで「ログインしたとき」に
最初に読み込まれる設定ファイルのひとつ。

ログイン直後の環境変数（PATHなど）を
まとめて設定する役割がある。

## なぜ確認するの？
LinuC102では
「どの設定ファイルが、いつ読まれるか」
がよく問われるため。

/etc/profile は
・ログインシェルのときだけ読まれる
という点が重要。

## 今回やったこと
- ls -l /etc/profile で権限を確認
- sed -n '1,120p' で先頭だけ表示
- grep で PATH / umask / HIST の痕跡を確認

## 分かったこと（所見）
- /etc/profile は root 管理のファイル
- 一般ユーザーは読むだけ
- PATH や HISTSIZE などの設定が含まれていた
- /etc/profile.d 配下の *.sh も読み込まれる

## メモ
- sed  の意味が最初分からなかった
- PS1 は /etc/profile には無いことも多い
- 「何が書いてあるか」より
  「いつ読まれるか」が大事
